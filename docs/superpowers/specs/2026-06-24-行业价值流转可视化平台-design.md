## 1. 背景与目标

**合并的原始需求**:

- `REQ-5752e7d1`:一个网站,展现一张大图,总体概括社会上各个行业(以卡片形式存在)之间的价值流转关系。点击各行业卡片,再次看到行业内部的价值流转关系。
- `REQ-5a2e865e`:首页是各个行业对整个社会的价值链分析,点击详情,是对行业内企业常见的痛点进行分析,并分析 AI 能力能给这些行业什么帮助。

**洞察**:两个 REQ 实际是同一项目的两个视图 — 首页(价值流转)+ 详情(痛点 + AI)。合并后用**统一架构**实现。

**MVP 范围**:**5 个**核心行业(农业/制造/金融/教育/医疗),个人本地使用,公开数据 + LLM 实时生成痛点。

- **高内聚低耦合**:`scraper/llm/storage/schema` 各自独立,通过 Protocol 解耦

- **可复用资产**:`llm/` 抽象层 + `scraper/` 抽象层独立登记 arch 平台

### 4.3 数据模型(Pydantic)

```python
class Industry(BaseModel):
    id: str  # "agriculture" / "manufacturing" / ...
    name: str  # 中文名
    color: str  # Sankey 配色

class ValueFlowNode(BaseModel):
    id: str
    label: str  # 子环节/上下游
    layer: int  # 0=起点, N=终点

class ValueFlowEdge(BaseModel):
    source: str  # node id
    target: str
    value: float  # 价值量(亿元)

class SankeyData(BaseModel):
    industries: list[Industry]
    nodes: list[ValueFlowNode]
    edges: list[ValueFlowEdge]

class PainPoint(BaseModel):
    title: str
    description: str
    severity: Literal["low", "medium", "high"]

class AIHelp(BaseModel):
    use_case: str
    capability: str
    example: str
    roi_estimate: str
```

## 5. 数据流

### 5.1 冷启动

```
[Browser 打开] → [React 启动] → [HomePage] → GET /api/industries
  ↓
[FastAPI] → check cache(SQLite)
  ├─ 命中 → 返回
  └─ 未命中 → 调 scraper/stats_gov.py.fetch_all()
              → 解析 → 写 cache → 返回 SankeyData
  ↓
[d3-sankey 渲染] → 用户看到首页
```

### 5.2 首页 → 二级详情

```
[用户点击"金融"] → 路由跳 /industry/finance
  ↓
[IndustryDetail 渲染] → 并行:
  ├─ GET /api/industry/finance → 子环节 Sankey(无 LLM,直接出)
  └─ POST /api/llm/generate(industry_id="finance")
  ↓
[FastAPI] → check LLM cache(key=hash(industry+provider+model+prompt))
  ├─ 命中 → 返回
  └─ 未命中 → 调 LLMProviderRegistry.get_active().generate(prompt)
              → 解析 → 写 cache → 流式返回(可选)
  ↓
[PainPanel] → Markdown 渲染痛点 + AI 帮助
```

### 5.3 缓存策略(SQLite)

```sql
-- 抓取结果:7 天 TTL
CREATE TABLE scrape_cache (
  key TEXT PRIMARY KEY,
  data JSON NOT NULL,
  scraped_at TIMESTAMP NOT NULL,
  expires_at TIMESTAMP NOT NULL
);

-- LLM 结果:30 天 TTL
CREATE TABLE llm_cache (
  key TEXT PRIMARY KEY,
  prompt TEXT NOT NULL,
  response JSON NOT NULL,
  generated_at TIMESTAMP NOT NULL,
  expires_at TIMESTAMP NOT NULL
);
```

**TTL 理由**:
- 抓取结果:7 天(行业数据基本不变)
- LLM 结果:30 天(同 prompt + provider + model 输出稳定,无需重复生成)

### 5.4 数据更新

- **手动触发**:Settings 加"刷新缓存"按钮
- **冷启动**:cache 过期时自动重新抓取
- **cron**(可选,留作未来):不进 MVP

## 6. LLM 多 Provider 架构

### 6.1 Provider 矩阵

| Provider | API 风格 | Base URL | Adapter 成本 |
|---|---|---|---|
| Claude | Anthropic 原生 | `https://api.anthropic.com` | 1 个文件 ~50 行 |
| OpenAI | OpenAI 原生 | `https://api.openai.com/v1` | 1 个文件 ~30 行 |
| DeepSeek | OpenAI 兼容 | `https://api.deepseek.com/v1` | 继承 openai_compatible |
| MiniMax | OpenAI 兼容 | `https://api.minimax.io/v1` | 继承 openai_compatible,支持 `enable_thinking` |
| Ollama | OpenAI 兼容 | `http://localhost:11434/v1` | 继承 openai_compatible(可选) |

### 6.2 OpenAI 兼容基类(关键)

```python
# llm/openai_compatible.py
class OpenAICompatibleProvider(LLMProviderProtocol):
    def __init__(self, base_url: str, api_key: str, default_model: str, extra_headers: dict | None = None):
        self.base_url = base_url
        self.api_key = api_key
        self.default_model = default_model
        self.extra_headers = extra_headers or {}
        self.client = httpx.AsyncClient(base_url=base_url, headers={"Authorization": f"Bearer {api_key}", **extra_headers})

    async def generate(self, prompt: str, model: str | None = None) -> str:
        payload = {"model": model or self.default_model, "messages": [{"role": "user", "content": prompt}]}
        resp = await self.client.post("/chat/completions", json=payload, timeout=60)
        resp.raise_for_status()
        return resp.json()["choices"][0]["message"]["content"]
```

子类(DeepSeek / MiniMax / Ollama)只需声明自己的 base_url / model / extra_headers,**零代码重复**。

### 6.3 MiniMax 特殊处理

- 唯一特有功能:`chat_template_kwargs: {enable_thinking: bool}`
- `llm/minimax.py` 子类重写 `generate`,加上该字段
- 不污染基类

### 6.4 Settings 切换

- 前端 Settings 加 provider 切换 tab(Claude/OpenAI/DeepSeek/MiniMax)
- 用户填 base_url + api_key + model
- 切换时调 `POST /api/settings/llm {provider, config}` 写入 settings.json
- `LLMProviderRegistry.reload()` 重新实例化 active provider

## 7. 错误处理与边界

### 7.1 LLM 成本保护(3 道防线)

```python
# 防线 1: 路由层限流(每分钟 10 次)
@router.post("/api/llm/generate", dependencies=[Depends(rate_limiter)])

# 防线 2: 每日 token 预算(Settings 可配,默认 100k/天)
class DailyBudget:
    limit: int = 100_000

# 防线 3: 单次 prompt token 上限
MAX_PROMPT_TOKENS = 4000
```

### 7.2 Prompt 注入防护

- Prompt 是**模板**而非**对话**,用户输入不直接进 prompt
- 行业数据从 Pydantic 模型取,无注入面
- LLM 输出严格 JSON 校验,失败自动重试 1 次

### 7.3 抓取失败降级链

```
抓取失败 → 重试 3 次 → 用旧缓存 → fallback 数据源 → 部分行业不可用 → 全失败返回错误
```

### 7.4 并发 Coalescer

- 同 industry_id 同时打开 5 个标签页 → 5 个 LLM 请求
- `LLMCoalescer` 用 `asyncio.Semaphore(3)` + 同 key 共享 in-flight promise
- 实际 LLM 只调 1 次,其他 4 个等待结果

### 7.5 API Key 安全

- 仅存 `~/.config/industry-value-flow/settings.json`,权限 600
- logger filter 屏蔽 `api_key` / `authorization`
- 前端不存 key,只调后端写入

### 7.6 边界场景

| 场景 | 行为 |
|---|---|
| 5 行业有 1 个抓不到 | 该卡片灰色"暂无数据",其他正常 |
| LLM 返回非 JSON | 重试 1 次(更严 prompt),失败用空数组 + log warn |
| LLM 输出偏离(全 high) | sanity check 标 `low_confidence` |
| Sankey 节点过多(>20) | 折叠低价值边(<总流量 1%) |
| LLM 全挂 | PainPanel 显示"AI 分析暂不可用",Sankey 正常 |

## 11. 已知风险与缓解

| 风险 | 缓解 |
|---|---|
| 国家统计局网页结构变化 → scraper 失效 | ScraperProtocol + 降级链 + 旧 cache 兜底 |
| LLM 限流/挂掉 → 用户体验差 | 缓存 + fallback 静态文案 + 清晰错误 |
| MiniMax enable_thinking 行为变化 | minimax.py 单独子文件,改一行不动其他 |
| d3-sankey 渲染性能 | 节点上限 20,自动折叠低价值边 |
| 双语言栈(TS+Py)维护成本 | 复用:scraper/llm/storage 都在 Py,前端只管展示 |
