# 接入真实数据 — Design Spec

**日期**: 2026-06-25
**作者**: Claude(架构对话辅助)+ 黄谦敏(架构师)
**前置 spec**: `docs/superpowers/specs/2026-06-24-行业价值流转可视化平台-design.md`(MVP 总设计)
**关联 REQ**:(本 spec 不直接关联 REQ,由后续实施计划关联)
**状态**: 待用户 review

---

## 1. 目标(Goals)

1. **主桑基图用真实数据**:首页 5 行业大桑基的 edge `value` 来自国家统计局 2024 年**分行业 GDP 增加值**(不再是占位的 100.0)。
2. **5 行业子行业桑基图全部接真实数据**:进入任一行业详情页,展示的是该行业下真实可查的子行业(种植业/医院/学校/制造业 11 大类)及其指标,**不再**是 `{ind_id}_root / _mid / _out` 占位。
3. **金融业改造为"资金来源 → 金融机构 → 资金运用"新叙事**:统计局不发布"银行/证券/保险"的 GDP 增加值,保留 5 行业原貌不可达,必须重新设计桑基结构。
4. **数据来源合规可追溯**:每张桑基图能告诉用户"数据从哪来、什么时候的、什么口径",前端 UI 标注。
5. **失败可降级**:接真实数据后任何抓取失败仍能 fallback 到 `industry_association.py`(占位骨架),不让系统挂掉。

---

## 2. 非目标(Non-Goals)

- ❌ **不做月/季度数据** — 只看 2024 年全年(单时间点快照)。
- ❌ **不做时间序列** — 不渲染历史曲线、不做趋势。
- ❌ **不做实时数据刷新** — 缓存 7 天,过期才重新抓取,无 cron。
- ❌ **不做数据预测 / 估算** — 拿不到的数据用 fallback,**不**用 LLM 编造。
- ❌ **不做多语言** — 全中文(原 MVP 设定)。
- ❌ **不做多年度对比** — 2024 年是唯一年度。

---

## 3. 5 行业子行业映射表(关键)

> **数据基准**:2024 年全年,数据发布日期 2025 年 1-3 月。
> **金额单位**:亿元(国内生产总值绝对值)。
> **未列出的小项**:汇总到"其他"节点(避免节点过多,符合原 spec "Sankey 节点 ≤ 20"边界)。

### 3.1 农业(GB/T 4754 代码 A)

**数据来源**:国家统计局《农业发展报告》及《国民经济核算》(年度数据)。2024 年农业 GDP 增加值: **94,431 亿元**(国家统计局 2025-01 公报)。
**抓取方式**:`cn-stats` 库 → 表 `A0201` 农林牧渔业分项增加值。

| 子节点 id | label | value(2024 亿元) | 占比 | 备注 |
|---|---|---:|---:|---|
| `agri_planting` | 种植业 | 60,212 | 63.8% | 谷物/油料/蔬菜/水果 |
| `agri_forestry` | 林业 | 6,604 | 7.0% | 林产/造林 |
| `agri_animal` | 畜牧业 | 20,517 | 21.7% | 猪牛羊禽 |
| `agri_fishery` | 渔业 | 7,098 | 7.5% | 海水/淡水 |
| `agri_service` | 农林牧渔服务业 | — | — | 数据**未单列**,归入"种植业"或"其他" |

**节点设计(3 层)**:
```
[种植业/林业/畜牧业/渔业] layer=0
        ↓
[农业 GDP 增加值合计 94,431]  layer=1 (mid)
        ↓
[消费端 / 工业原料]            layer=2 (out, 共享 consumer)
```

**真实来源 URL**:
- 国家统计局: https://data.stats.gov.cn/easyquery.htm?cn=C01(分行业增加值)
- 农业农村部: http://www.moa.gov.cn/(细分行业不公开,仅总数)

**数据可得性**:**良**。种植业/畜牧业/渔业数据稳定发布,林业偶有合并到"其他"。

### 3.3 金融业(GB/T 4754 代码 J)— 难点

**关键发现**:国家统计局**不公布**分金融子行业(银行/证券/保险/基金)的 GDP 增加值。增加值总量可查(2024 ≈ **98,800 亿元**),但"银行/证券/保险"细分仅在央行/银保监会/证监会各自的**监管统计**中,且单位是"资产/负债/保费"而非"增加值"。

**新叙事**:**资金来源 → 金融机构 → 资金运用**(资金流量表的简化版,非生产视角)。
**数据来源**:
- 中国人民银行《金融统计数据报告》(月度,2024-12): https://www.pbc.gov.cn/
- 国家金融监督管理总局《银行业保险业统计信息》: https://www.nfra.gov.cn/
- 中国证监会《证券期货市场统计月报》: http://www.csrc.gov.cn/

| 子节点 id | label | value(2024 万亿元) | 来源 |
|---|---|---:|---|
| **layer 0 — 资金来源** | | | |
| `fin_src_household_dep` | 住户存款 | 152.3 | 央行《社融存量》 |
| `fin_src_corp_dep` | 企业存款 | 78.2 | 央行 |
| `fin_src_fx` | 外汇占款 | — | 暂缺 |
| **layer 1 — 金融机构** | | | |
| `fin_inst_bank` | 银行业金融机构 | 372.0(总资产) | 银保监会 |
| `fin_inst_sec` | 证券业 | 13.5(总资产) | 证监会 |
| `fin_inst_ins` | 保险业 | 33.3(总资产) | 银保监会 |
| **layer 2 — 资金运用** | | | |
| `fin_use_loan` | 各项贷款 | 258.4 | 央行 |
| `fin_use_premium` | 保费收入 | 5.7 | 银保监会 |
| `fin_use_raised` | 证券承销筹资 | 4.5(2024 IPO+增发) | 证监会 |

> **单位**:金融业改为"万亿元",**与农业/制造业的"亿元"不同**。需在 UI 上明确标注单位(每个行业单独一行 footer)。

**节点设计**:
```
[住户存款 / 企业存款 / 其他]   layer=0 (资金来源)
        ↓
[银行 / 证券 / 保险]            layer=1 (金融机构)
        ↓
[贷款 / 承保额 / 筹资]          layer=2 (资金运用)
```

**前端展示差异**:与另 4 行业"投入-加工-产出"叙事**完全不同的桑基布局**。需要在 `Sankey.tsx`/`MiniSankey.tsx` 保持 layout 通用,但 `layer=0/1/2` 标签按行业切换。

## 4. Schema 变更

### 4.1 Pydantic 模型

**文件**:`backend/schema/sankey.py`

```python
class SankeyData(BaseModel):
    industries: list[Industry]
    nodes: list[ValueFlowNode]
    edges: list[ValueFlowEdge]
    source: str | None = None   # 新增:数据来源说明,例 "国家统计局 2024 年公报"
    year: int | None = None     # 新增:数据年度,默认 2024
    unit: str = "亿元"          # 新增:单位,例 "亿元" / "万人" / "万亿元"
```

### 4.2 Edge 校验放宽

```python
class ValueFlowEdge(BaseModel):
    source: str
    target: str
    value: float = Field(ge=0.0)   # 原: gt=0.0  → 改为 ge=0.0 允许 0
```

**理由**:真实数据 0 值会出现(如某小类当年未单列),0 是合法值;同时 `value=0` 的 edge 在前端 d3-sankey 应被自动跳过(防 NaN)。

### 4.3 节点 id 命名

**旧**: `{ind_id}_root / _mid / _out`(占位)
**新**: 具体子行业 id(见 §3 表格),如 `agri_planting`、`med_hospital`、`fin_inst_bank`

**路由层兼容**:`GET /api/industry/{ind_id}` 行为不变,只是返回的 `nodes[].id` 更具体。

**前端影响**:`Sankey.tsx` 第 24 行 `n.id.split("_")[0]` 用于推断行业配色 — 需改为按 `n.id` 前缀匹配(`agri_*` → `#4a90e2`),见 §7.1。

### 4.4 TypeScript 类型同步

**文件**:`frontend/src/types/api.ts`

```ts
export type SankeyData = {
  industries: Industry[];
  nodes: ValueFlowNode[];
  edges: ValueFlowEdge[];
  source: string | null;   // 新增
  year: number | null;     // 新增
  unit: string;            // 新增
};
```

### 5.4 SSL verify=False 风险

**风险**:cn-stats 默认 `verify_ssl=False`(国内老旧服务器证书链不全)。
**缓解**:
1. 适配层显式 `verify_ssl=False` + log warn
2. 配置项 `IVF_STATS_VERIFY_SSL`(默认 `false`)
3. 文档明确"本地运行风险自担"
4. **不**部署到公网,本地单人使用,符合 MVP 设定

### 5.5 异常与降级

**保留**:`routes/industries.py` / `routes/industry.py` 的 try/except + fallback `IndustryAssociationScraper`,逻辑不动。
**升级**:`industry_association.py` 改为**部分真实 + 占位**混合:
- 农业 / 制造业:cn-stats 失败时,`industry_association.py` 返回**该年统计公报的硬编码数值**(如农业 2024 增加值 94,431 亿元)
- 金融 / 教育 / 医疗:cn-stats 完全不覆盖,直接返回**该子行业的真实数值硬编码**

**这意味着 `industry_association.py` 不再是纯占位,而是"快照式缓存"**。

## 6. 缓存策略

### 6.1 缓存 key 版本化

**旧**:
- `industries:all:v1`
- `industry:{ind_id}:v1`

**新**(按年份 + schema 版本):
- `stats_gov:industries:{year}:v2`
- `stats_gov:industry:{ind_id}:{year}:v2`
- `assoc:industries:{year}:v2`(fallback)

**`v2` 含义**:与原 v1 schema 不兼容(加 `source/year/unit` 字段)。**老 cache 必须清掉**(可用 `Cache.delete` 显式清,或等 7 天 TTL 自然过期)。

**建议**:实施时**加 cache migration**:
```python
# 启动时清掉 v1
for key in [r[0] for r in conn.execute("SELECT key FROM scrape_cache WHERE key LIKE '%:v1'")]:
    cache.delete(key)
```

### 6.2 TTL

- 抓取结果:**7 天**(不变,符合"7 天数据基本不变"假设)
- LLM 结果:30 天(不动,非本 spec 范围)

### 6.3 命中率优化

- 首页 `industries:all:v2` 一次抓,5 个行业详情共享子行业数据
- 教育 / 医疗 / 金融在 cn-stats 之外,直接读 `industry_association.py` 硬编码(零网络,**100% 命中**)
- 仅农业 / 制造业 + 主桑基合计会触发真实抓取

## 7. 前端变更

### 7.1 节点配色按子行业继承

**问题**:原 `Sankey.tsx` 第 24 行 `n.id.split("_")[0]` 在新 id 规则下失效:
- `agri_planting.split("_")[0]` → `"agri"` → 查 `industries` 找到 `#4a90e2` 正确
- `med_hospital.split("_")[0]` → `"med"` → 查 `industries` **没有** `med` → 走 fallback `"#888"`

**修复**:`Sankey.tsx` 改用**子行业调色板**(每个行业一个子色阶):

```ts
// 思路(伪代码,实施时实现)
const subPalette: Record<string, string[]> = {
  agriculture: ["#4a90e2", "#5fa3e8", "#75b6ee", "#8bc9f4", "#a1dcfa"],
  manufacturing: ["#e94560", "#ed5e76", "#f1778c", "#f590a2", "#f9a9b8"],
  finance: ["#50c878", "#6ad28a", "#84dc9c", "#9ee6ae", "#b8f0c0"],
  education: ["#ffd700", "#ffe14d", "#ffeb7a", "#fff5a7", "#ffffd4"],
  healthcare: ["#9b59b6", "#a96dc1", "#b781cc", "#c595d7", "#d3a9e2"],
};
const colorFor = (nodeId: string): string => {
  const prefix = nodeId.split("_")[0];  // "agri" / "mfg" / "fin" / "edu" / "med"
  const palette = subPalette[fullNameByPrefix[prefix]];
  const idx = nodeId.split("_").length - 1;  // 0 = 第一子
  return palette?.[idx] ?? "#888";
};
```

**详细**:`miniSankey.tsx` 同步改。

## 10. 风险与缓解

| # | 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|---|
| R1 | cn-stats 包突然不维护 / API 改版 | 中 | 主桑基图全挂 | 已 fallback `industry_association.py` + 真实硬编码;后续可换包只改适配层 |

| R4 | 网络抖动 scraper 失败 | 高 | 单次请求失败 | 已有 3 次重试 + 旧 cache 兜底 + fallback 链 |
| R5 | 金融业"万亿元"单位与其他 4 行业"亿元"不一致 | 高 | 用户困惑 | UI footer 显式标注单位(§7.2);不加自动换算(避免精确度丢失) |

| R7 | cn-stats 默认 `verify_ssl=False` 的安全风险 | 中 | 中间人攻击 | 仅本地运行,不暴露公网;配置项 + log warn |
| R8 | 子行业节点 > 20(d3-sankey 上限) | 低 | 渲染拥挤 | 制造业 11 大类 + 其他 = 12 节点,在限内;金融 9 节点,安全;若超,折叠 < 总流量 1% 的边 |
| R9 | `source/year/unit` 字段全为 None 时前端崩 | 低 | 白屏 | Pydantic 字段默认 `None / ""`,TypeScript 类型允许 `null`,组件做条件渲染 |
| R10 | 旧 cache v1 与新 schema v2 不兼容 | 中 | 用户看到旧数据 | 启动时主动清 v1(§6.1) |
