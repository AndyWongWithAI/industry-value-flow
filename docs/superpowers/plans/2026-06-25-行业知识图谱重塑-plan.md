# 行业知识图谱重塑 Implementation Plan (简短版)

> **基线**:基于已批准 spec `docs/superpowers/specs/2026-06-25-行业知识图谱重塑-design.md`
> **目标**:把"行业价值流转可视化平台"重塑为"行业知识图谱"

**Goal:** LLM 初始化生成 100+ 行业间相互支撑关系知识图谱,前端 react-flow 渲染,明色系企业风。

**Architecture:** 启动时 LLM 生成图 → 存 SQLite → react-flow force/dagre 布局 → 用户点击节点/边调 LLM 实时解释。

**Tech Stack:** FastAPI + Pydantic v2 + SQLite + Vite + React + TS + react-flow + dagre + 现有 LLM 抽象(5 provider)。

## 关键全局约束(从 spec 直接 copy,所有任务隐含遵守)

- **节点**:GB/T 4754-2017 中类(100+ 项),LLM 生成
- **边**:定性关系(provide/rely_on/service/consume),weight 1-5,LLM 生成
- **可视化**:react-flow + dagre 布局(不持久化节点位置)
- **数据**:纯 LLM 驱动初始化,存 SQLite,cache key `graph:v1:{llm_config_hash}`,TTL 7 天
- **LLM 失败**:不缓存,启动报错;前端白屏 + "请配置 LLM" 提示
- **LLM 角色**:初始化 + 用户查询解释(中间不参与)
- **UI**:明色系企业风(白底 + 品牌蓝 #1A4D8F + 思源黑体 + 留白 24-32px)
- **持久资产**:LLM provider 抽象(5 provider)、Settings 页、SQLite 基础设施、pytest + Playwright
- **删除资产**:SankeyData / Sankey.tsx / cn-stats / industry_association.py / stats_gov.py / Footer.tsx / 5 行业枚举 / 子色阶 / E2E-004/005/006

## 风险与缓解

| 风险 | 缓解 |
|---|---|
| LLM 生成节点不覆盖所有 GB/T 4754 中类 | 不强制,生成的就行 |
| LLM 生成"幻觉关系" | validator 校验 source/target 必须存在 + relation_type 在枚举内 |
| LLM 启动失败 | 7 天坏数据比启动慢更糟 → 失败不缓存,每次启动重试 |
| 100+ 节点 force 性能 | dagre 比 force stable,react-flow 100+ 节点性能足够 |
| 用户切 provider → 重新初始化 | llm_config_hash 变化触发 |
| LLM JSON 格式错误 | 严格 schema 校验 + 1 次重试 + 失败报错 |

## 不做(防止 scope creep)

- 用户系统 / 权限
- 关系人工校验流程
- 导出图片 / PDF(留 backlog)
- 多进程 / 多用户并发(本地单人)
